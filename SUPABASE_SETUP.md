# 🗄️ Supabase Integration Setup Guide\n\n## Step 1: Create Supabase Project\n\n1. Go to [supabase.com](https://supabase.com)\n
2. Click \"Start your project\" and sign up/login\n
3. Click \"New Project\"\n
4. Fill in:\n   - **Name**: `urban-stress-ml`\n   - **Database Password**: Create a strong password\n   - **Region**: Choose closest to your users\n
5. Click \"Create new project\"\n
6. Wait for project setup (2-3 minutes)\n\n## 
Step 2: Get Your Supabase Credentials\n\n1. In your Supabase dashboard, go to **Settings** → **API**\n2. Copy these values:\n   - **Project URL** (looks like: `https://xxxxx.supabase.co`)\n   - **anon public key** (starts with `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...`)\n\n## Step 3: Set Up Database Schema\n\n1. In Supabase dashboard, go to **SQL Editor**\n2. Click \"New Query\"\n3. Copy and paste the contents of `supabase_schema.sql`\n4. Click **Run** to create the tables\n\n## Step 4: Configure Environment Variables\n\n1. Copy `.env.example` to `.env`:\n   ```bash\n   cp .env.example .env\n   ```\n\n2. Edit `.env` file with your Supabase credentials:\n   ```bash\n   SUPABASE_URL=https://your-project-id.supabase.co\n   SUPABASE_ANON_KEY=your_anon_key_here\n   FLASK_ENV=development\n   FLASK_DEBUG=1\n   ```\n\n## Step 5: Install Dependencies\n\n```bash\ncd backend\npip install -r requirements.txt\n```\n\n## Step 6: Test the Connection\n\n1. Generate ML model if not exists:\n   ```bash\n   python ml_pipeline_urban.py\n   ```\n\n2. Start the Flask app:\n   ```bash\n   cd backend\n   python app.py\n   ```\n\n3. Test the API endpoints:\n   ```bash\n   # Test model info\n   curl http://localhost:5000/info\n   \n   # Test prediction (this will log to Supabase)\n   curl -X POST http://localhost:5000/predict \\\n     -H \"Content-Type: application/json\" \\\n     -d '{\n       \"crosswalk_wait_sec\": 35.0,\n       \"litter_density_per_100m2\": 5.0,\n       \"midnight_noise_db\": 58.0,\n       \"atm_queue_length\": 2.0,\n       \"vacant_storefronts_pct\": 10.0,\n       \"dog_walk_freq_per_hr\": 1.0,\n       \"graffiti_turnover_days\": 10.0\n     }'\n   \n   # Test statistics\n   curl http://localhost:5000/stats\n   \n   # Test recent predictions\n   curl http://localhost:5000/recent\n   ```\n\n## Step 7: Verify Data in Supabase\n\n1. Go to Supabase dashboard → **Table Editor**\n2. Click on `predictions` table\n3. You should see your test prediction logged\n\n## 🔧 Available API Endpoints\n\n| Endpoint | Method | Description |\n|----------|--------|-------------|\n| `/info` | GET | Get model information |\n| `/predict` | POST | Make prediction (logs to Supabase) |\n| `/stats` | GET | Get prediction statistics |\n| `/recent` | GET | Get recent predictions |\n\n## 📊 Supabase Dashboard Features\n\n- **Real-time data**: See predictions as they come in\n- **SQL queries**: Run custom analytics queries\n- **API auto-generation**: REST and GraphQL APIs\n- **Authentication**: Built-in user management\n- **Storage**: File uploads if needed\n\n## 🔒 Security Notes\n\n- Row Level Security (RLS) is enabled\n- Anonymous users can read prediction stats\n- Authenticated users have full access\n- API keys are environment-specific\n\n## 🚀 Production Deployment\n\nFor production, set these environment variables in your deployment platform:\n\n```bash\nSUPABASE_URL=https://your-project-id.supabase.co\nSUPABASE_ANON_KEY=your_production_anon_key\nFLASK_ENV=production\nFLASK_DEBUG=0\n```\n\n## 📈 Monitoring & Analytics\n\nWith Supabase connected, you can:\n- Track prediction patterns over time\n- Monitor model performance\n- Analyze urban stress trends\n- Create custom dashboards\n- Set up alerts for high stress predictions\n\n---\n\n**✅ Your Urban Stress ML model is now connected to Supabase!**
